## Prompt 1 - Ask AI for a PR Draft

Review the current branch changes and draft a pull request description.

Use the git diff, changed files, and commit history as context.

Create a pull request draft with these sections:

1. Title
2. Summary
3. Why This Change Was Needed
4. What Changed
5. Validation Performed
6. Security and Privacy Considerations
7. Modernization or Maintainability Notes
8. Reviewer Focus Areas
9. Known Risks or Follow-Up Work

Rules:
- Do not invent issue numbers.
- Do not claim tests passed unless the validation notes confirm it.
- Do not claim security approval or production readiness.
- Keep the wording professional and concise.
- Make the reviewer focus areas specific and actionable.
